additive="sum" vs additive="replace"

1. replace + single type — ✓ works

Three type="translate" on same <rect>. Each replaces previous. from matches frozen state → no jump. Rect returns to origin.

2. replace + two types on same element — ✗ broken

translate + scale on same <g>, both replace. Scale (last in DOM) wins — translate is lost. Rect scales 2× but stays near origin. Did not translate 80px right.

3. sum + two types on same element — ✓ works

Same elements, additive="sum". Both compose: result = translate × scale(2). Rect is 2× larger AND translated.

4. sum + non-identity from — ✗ double-accumulation

Segment 1 freezes at x=60. Segment 2: from="60 0" to="120 0". With sum: frozen(60) + (60→120) = 120→180. Rect overshoots to x=200 instead of x=140.

5. sum + identity from — ✓ correct

Same segments, but segment 2: from="0 0" to="60 0" (identity). frozen(60) + (0→60) = 60→120. Lands on the green dot.
